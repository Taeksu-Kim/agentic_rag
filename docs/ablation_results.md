## 해석 (2026-07-18 분석)

**리랭크 축**: 0.6B 크로스인코더가 스위트스팟. 9B LLM 리랭크가 MRR +2pp 더 높지만
비용이 ~30배(0.16s vs 4.3s 순차). sparse 단독은 리랭커로도 회생 불가(1단계가 못
가져온 문서는 리랭커가 못 살린다).

**검색 축**: 리랭크 없이는 dense가 hybrid를 이긴다 — 형태소 분석 없는 한국어
BM25(R@8 0.238)가 RRF를 오염. 리랭커를 얹으면 hybrid가 회복(후보 풀 확장 기여,
노이즈는 리랭커가 필터). dense+ce와 hybrid+ce는 사실상 동급.

**에이전트 축 (정직한 네거티브 결과)**: react 반복이 이 평가셋에선 1-shot
hybrid+ce를 넘지 못한다 (R@8 0.515 vs 0.572 @s6). 정밀도 각도(P@1/P@3)로 봐도
동일하게 열세. 원인 두 가지:
1. **law_names 필터 오용**: 9B가 존재하지 않는 법령명 표기로 필터를 걸어 0건 →
   s2에선 만회 불가(빈손 17%). 스텝 예산이 늘면 회복(빈손 20→12→9개)하지만
   비용이 latency 25→112s로 폭증.
2. **평가셋 특성**: 질의회시 질문은 이미 법률 용어로 정제된 문장이라 에이전트의
   핵심 무기인 리라이팅의 이득 여지가 작다. 일상어 질문(예: "육아휴직 때 연차
   나와요?" — 라이브 검증에서 리라이팅으로 top-30 밖→1위)에서는 반대 결과가
   예상됨 → 일상어 변형 평가셋이 후속 과제.

**청킹 불필요 재확인 (데이터 근거)**: 정답 조문 길이별 top-8 회수율(hybrid+ce)이
~500자 0.445 < 500~1000자 0.587 < 1000~2000자 0.606 — miss는 **긴 조문이 아니라
짧은 조문**(정의·준용 조항류, 저정보)에 몰려 있다. 임베딩 절단이 문제가 아니므로
항 단위 청킹은 이득이 없다. (2000자+ 라벨은 n=6뿐이라 통계 무의미.)

**재현**: `PYTHONPATH=. python scripts/run_ablations.py retrieval|agent|report`
(재개 가능; per-query 결과는 data/eval/ablation/*.jsonl). 병렬(--workers 3) 실행
시 latency는 서버 큐잉 포함 참고치(순차 실측: hybrid+ce 0.16s, hybrid+llm 4.3s).

## 수정판(v3) 검증 — 타깃 재측정 (2026-07-18)

v1 실패 분석(위 "에이전트 축") 후 세 가지를 수정하고, 실패를 겨냥한 최소
부분집합으로 재측정했다. 수정: ① evidence = LLM 선택 상위 5 + 세션 고점수 채움
(큐레이션 실패 방어) ② 법령명 24종 목록을 툴 설명에 명시 + 공백 정규화 + 복수
지정 허용 ③ 0건 필터 자동 해제 (+ 프레임워크 JSON 추출 `group(1)` 버그픽스).

**A. 필터 가드** (v1 s2가 빈손이던 20쿼리):

| | 빈손 | R@8 | MRR |
|---|---|---|---|
| v1 | 20/20 | 0.000 | 0.000 |
| v3 | **0/20** | 0.408 | 0.485 |

**B. 큐레이션 수정** (멀티라벨 62쿼리, s4 페어드):

| | R@8 | MRR |
|---|---|---|
| s4 v1 | 0.446 | 0.616 |
| s4 v3 | **0.505** (+5.9pp) | **0.697** (+8.1pp) |
| 1-shot hybrid+ce | 0.558 | 0.697 |

격차 절반 회복, MRR은 1-shot과 동률. 빈손 62중 1, 에러 0.

**C. 일상어(현실) 평가셋** — 질의회시 120개를 9B로 "직원이 챗봇에 물어볼 1~2문장
구어체"로 변형(라벨 유지, 정상 변형 101개만 채점):

| (동일 101쿼리) | R@8 | MRR |
|---|---|---|
| hybrid (일상어) | 0.409 | 0.429 |
| **hybrid+ce (일상어)** | **0.543** | **0.570** |
| react-s4 v3 (일상어) | 0.449 | 0.484 |
| hybrid+ce (원본 질문) | 0.557 | 0.579 |

**판정 — 리라이팅 가설 기각**: 일상어 입력에서도 에이전트는 1-shot+CE를 넘지
못했다 (페어드 R@8 승 11 / 패 24 / 무 66). 이유가 표에 있다: **CE 리랭커가
어휘 갭을 이미 거의 흡수한다** (일상어 전환 시 hybrid는 −13pp지만 hybrid+ce는
−1.4pp). 리랭커가 강하면 리라이팅의 이득 여지 자체가 사라지고, 에이전트의
쟁점별 좁은 쿼리는 오히려 원 질문 통짜 검색의 커버리지를 잃는다.

**최종 결론**: 단일 코퍼스 조문 검색에서는 **hybrid + 0.6B 크로스인코더 1-shot이
품질/비용 최적점**이다. 에이전틱 반복의 가치는 검색 품질이 아니라 해석가능성
(쟁점 분해 트레이스, 조문 인용 답변)과 멀티소스 확장에 있다. 남은 개선 카드:
에이전트가 리라이팅 쿼리만 쓰지 말고 **원 질문 그대로의 검색을 항상 1회 포함**
(1-shot 이상을 구조적으로 보장) — 미측정, 후속 과제.

**부록 — 리라이팅 단독 효과 분리 (핀포인트, 일상어 101쿼리)**: 루프를 빼고
"원문 그대로 vs 9B 리라이팅 1회", 둘 다 hybrid+ce로 검색:

| | R@8 |
|---|---|
| 원문(일상어) 그대로 | **0.543** |
| 9B 리라이팅 1회 후 | 0.520 |

페어드 승 8 / 패 12 / **무 81**. 리라이팅은 대부분 무해무익(81 동률)이고 순효과는
미세 손해 — 리랭커가 있는 스택에선 어휘 갭이 이미 해소되어 리라이팅이 **불필요**
해지고, 가끔 질문의 디테일을 떨어뜨려 손해를 본다. 리라이팅이 값을 하는 조건은
리랭커 없는 약한 스택(일상어에서 hybrid −13pp)으로 추정 — 미측정.

## 레버 실장 결과 — union(합집합 리라이팅) · struct(조문 그래프 확장) (2026-07-18)

미스 유형 분해(부수 인용 73.5% / 저정보 조문 4배 농축 / 어휘 갭)에서 도출한
레버 2개를 실장하고 전체 재측정:

**격식 701 전체 (R@8 사다리):**

| 단계 | R@8 | MRR | p50 |
|---|---|---|---|
| hybrid+ce (기존 최고) | 0.575 | 0.584 | 0.16s |
| **union+ce** (+원문·리라이팅 합집합 풀) | 0.610 | 0.594 | 2.9s |
| **struct+ce** (+조문 그래프 동반 확장) | **0.633** | **0.603** | 3.4s |

- 주인용 조문 기준 R@8: **0.719** (struct, 격식 701)
- struct의 MRR은 30배 비싼 9B LLM 리랭크(0.603)와 동률.
- 그래프: "법 제N조" 역참조 정규식으로 위임 1,508 + 벌칙 324 + 교차 84 =
  1,916엣지 (LLM 0회, `retriever/edges.py`). 동반 조문도 CE 점수로 경쟁시켜
  무관 동반은 자동 탈락.

**일상어 120:** union+ce 0.589 / struct+ce 0.576 (MRR 0.581/0.565) — 일상어에선
그래프 확장이 이득 없음(−1.3pp, n=120 노이즈 범위 내). 동반 조문이 top-8에서
정답을 밀어내는 케이스가 있는 것으로 보이며 expand_top/경쟁 방식 튜닝 여지.
격식(회답 인용 세트가 동반 조문을 많이 포함)에서만 확실한 이득 — **엣지 확장의
가치는 채점 기준이 '완결 인용 세트'일 때 극대화**된다는 정직한 주석.

**시딩 에이전트**: react-s4에 "원 질문 검색 1회 상시 시딩"을 넣어 멀티라벨 62
페어드에서 R@8 0.446(v1)→0.505(v3)→**0.553(시딩)** — 1-shot(0.558)과 사실상
동률. 에이전트 하한 보장 확인.

**다음 레버(미실장, 설계 확정)**: 리랭커 점수 τ 필터(공짜) → 조건부 세트 판정
1회(LLM이 압축 뷰로 커버리지·재계획 — CE가 못 하는 세트 수준 판단만 담당) →
전문 정독은 플래그된 청크만 에스컬레이션. 관측 재설계(상위 2건 본문 600자 —
위임 73%/교차참조 92%/단서 90% 가시) 포함.

## kiwi 형태소 BM25 + 채점 교정 (2026-07-19)

**형태소 BM25** (`retriever/kiwi_bm25.py` — kiwi 내용 형태소 위 BM25 직접 계산,
컬렉션 `statutes_kiwi`): sparse 축이 부활하며 사다리 전체가 이동.

| arm (격식 701) | 기존 R@8 | kiwi R@8 |
|---|---|---|
| sparse | 0.238 | **0.507** (2.1배) |
| hybrid | 0.487 | **0.570** (dense 0.536 첫 역전) |
| hybrid+ce | 0.575 | **0.616** (LLM 0회로 union 0.610 추월) |
| struct+ce | 0.633 | **0.640** |

상층부로 갈수록 이득이 줄어드는 것(+8.3 → +4.1 → +0.7pp)은 union·엣지가 구제하던
미스와 kiwi가 구제하는 미스가 겹치기 때문 — 저비용 구성(hybrid+ce, 0.14s)의
위상이 크게 상승. ⚠ 임베딩 재색인 시 초장문 조문은 클라이언트 3,500자 절단
(vLLM pooling의 truncate_prompt_tokens는 임베딩 엔드포인트에서도 무한 대기 —
/score와 동일 실패 모드, 3번째 실측).

**채점 교정 — "검색 문제"의 상당분은 "라벨 문제"였다**: 확답형 주인용 미스
44개 중 표본 6개를 정독한 결과 4~5개가 검색 실패가 아니라 **labels[0](회답의 첫
인용)이 보일러플레이트(목적·적용범위, 11.2%)거나 교차 인용 앵커**인 인공물
(예: "임금을 통화 외로 지급 가능?"에 검색은 근기법 43조(통화불 원칙)를 1위로
냈는데 라벨은 1조(목적)). 9B가 회답을 읽고 핵심 근거 조문을 재선택하니
(`scripts/relabel_primary.py`, 47/188 교체) 최종 헤드라인:

| struct+ce(kiwi), 확답형 188 | |
|---|---|
| **핵심 조문 R@8 / R@3** | **0.782 / 0.686** |
| 실질 라벨 any-hit@8 | 0.862 |

**남은 진성 실패(~14~22%)의 얼굴**: 질문 어휘와 조문 어휘의 진성 갭 (예:
"KCs 표시된 제품만 사용 가능?" vs 제87조 "제조·수입·양도·대여·사용 금지") —
임베더 체급(0.6B) 업그레이드가 다음 레버. 그 외 교차 법령 2차 인용은 조건부
세트 판정 라운드(설계 확정, 미실장) 몫.

## 퓨샷 리라이터 — "리라이터가 약한 탓" 가설 기각 (2026-07-19)

앞선 부록에서 "리랭커가 있으면 리라이팅은 무해무익"이 나왔지만, 반론이 하나
남아 있었다: **리라이터가 제로샷이라 약했던 것 아닌가?** 이를 제거하려고
`retriever/rewrite.py`에 창작 퓨샷 3개(일상어→규범어 명사구 변환 시범, 평가셋
표현 미사용)를 넣어 리라이터를 강화하고, kiwi 컬렉션에서 재측정했다. 라이브
확인: "육아휴직 쓰고 오면 연차 그대로 생기나요"→"육아휴직 사용 후 연차유지",
"입사 3개월 퇴직금"→"입사 3개월 퇴직금 지급 요건" 등 **질적으로 더 깨끗한
법률 명사구**를 뽑는다(퓨샷 내용 앵무새 없음, 새 주제로 일반화 확인).

| | R@8 base→퓨샷 | MRR | paired 승/패 |
|---|---|---|---|
| union+ce (일상어 120) | 0.598 → 0.592 | .573→.568 | 3 / 3 |
| struct+ce (일상어 120) | 0.613 → 0.601 | .585→.577 | 1 / 1 (120중 2건만 변동) |
| union+ce (격식 701) | 0.627 → 0.628 | .596→.601 | 9 / 9 |
| struct+ce (격식 701) | 0.640 → 0.637 | .604→.598 | 4 / 8 |

**판정**: 전부 노이즈 범위 — 리라이터를 눈에 띄게 개선해도 검색 지표는
움직이지 않는다(일상어 struct는 120건 중 **2건**만 순위가 바뀜). 즉 병목은
리라이터 품질이 아니라 **CE 리랭커가 이미 어휘 갭을 흡수**하는 구조라, 리라이팅
쪽에 남은 헤드룸 자체가 없다. "리라이터가 약했던 탓" 대안 설명을 제거해
리랭커-지배 결론이 강화된다. (승리 구성 hybrid+ce 0.616은 애초에 리라이팅을
쓰지도 않는다.) 퓨샷은 트레이스 가독성·union 풀 다양성 이득이 있어 **유지**하되,
검색 지표 기여는 없다고 정직히 기록. 다음 실질 레버는 여전히 **임베더 체급**.
